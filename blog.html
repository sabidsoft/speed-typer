Difference Between Local Storage and Session Storage

  1. Local storage is especially useful for storing data that you want to persist even if the user closes the browser, such as preferences or settings. Session storage is useful for storing data that is sensitive, such as login credentials.
  2. In local storage, the stored data is not deleted when the browser is closed. In session storage, the data is stored only for the session and will be deleted when the browser is closed.
  3. In local storage, the data is stored on the browser and system. In session storage, the data is stored on the browser only.
  4. The storage capacity of local storage is 5MB/10MB. The storage capacity of session storage is 5MB.

Difference Between Global Scope and Block Scope

  1. Variables declared Globally have Global Scope. Variables declared within a JavaScript curly braces { } become block scope (without var keyword).
  2. In global scope, variables declared with var, let and const keyword. In block scope, variables declared with let and const keyword.
  3. Global variables can be accessed from anywhere in a JavaScript program. In block scope, variables declared inside a { } block cannot be accessed from outside the block

Event Loop

An event loop is something that pulls stuff out of the queue and places it onto the function execution stack whenever the function stack becomes empty.The event loop is the secret by which JavaScript gives us an illusion of being multithreaded even though it is single-threaded. Event loop is an endless loop, which waits for tasks, executes them and then sleeps until it receives more tasks. The event loop executes tasks from the event queue only when the call stack is empty i.e. there is no ongoing task. The event loop allows us to use callbacks and promises. The event loop executes the tasks starting from the oldest first.

How to Get Undefined

  1. When we declare a variable and don't initialize it to a value, the variable will have a value of undefined.
  2. If we access a non-existing property of an object, we will get undefined.
  3. A function that doesn't have a return statement implicitly returns undefined
  4. When we access an array element that is out-of-bounds, we will get the undefined value.
